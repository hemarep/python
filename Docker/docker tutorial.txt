==============
==== 1
==============
First, clone a repository
The Getting Started project is a simple GitHub repository
which contains everything you need to build an image and run it as a container.

Clone the repository by running Git in a container.

arun@macbook:~$ docker run --name repo alpine/git clone https://github.com/docker/getting-started.git
arun@macbook:~$ docker cp repo:/git/getting-started/ .


You can also type the command directly in a command line interface.
==============
==== 2
==============
Now, build the image
A Docker image is a private file system just for your container.
It provides all the files and code your container needs.

arun@macbook:~$ cd getting-started
arun@macbook:~/getting-started$ docker build -t docker101tutorial .
==============
==== 3
==============
Run your first container
Start a container based on the image you built in the previous step.
Running a container launches your application with private resources, securely isolated from the rest of your machine.
arun@macbook:~/getting-started$ docker run -d -p 80:80 --name docker-tutorial docker101tutorial


01tutorial hema2python/docker101tutorial
"docker push" requires exactly 1 argument.
See 'docker push --help'.

Usage:  docker push [OPTIONS] NAME[:TAG]

==============
==== 4
==============
Now save and share your image
Save and share your image on Docker Hub to enable other users to easily download and run the image on any destination machine.

See what you've saved on Hub
https://hub.docker.com/repositories hema14it@gmail.com python123$

arun@macbook:~/getting-started$ docker push hema2python/docker101tutorialdocker tag docker1
